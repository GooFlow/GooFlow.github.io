Event Hooks

Toolbar Button Events

he following is the event definition (virtual function) that is triggered when the toolbar button is clicked. Only when the head bar is set when the GooFlow instance is initialized, these event hooks exist. Because you can directly this access the GooFlow instance itself, there is no pass parameter; users can redefine themselves.

In addition, the user can also use HeadBtnEvent interface to extend the click event of another custom head button through the built-in interface of the GooFlow instance.

How to use these event hooks: 
    instance.myEvent = function(){
        this;  //GooFlow instance
        //TODO: The realization of event response
    };

Event when new flowchart button is clicked.

Event when open flowchart button is clicked.

this.onBtnSaveClick() Return: void

Event when save flowchart button is clicked.

this.onFreshClick() Return: void

Event when refresh flowchart button is clicked.

this.onPrintClick() Return: void

Event when printer button is clicked.

Flowchart Events

The following bindings are some of the operational events of various flow chart elements, which users can redefine themselves. These events can access directly the GooFlow instance itself through this; and the methods triggered by these events, if returned false, it will prevent the default operation or event from happening.

How to use these event hooks: 
    instance.myEvent = function( args[] ){
        this;  //GooFlow instance
        //TODO: The realization of event response
        return bool; //Return to false to prevent default events, or return to other values to allow default events to occur
    };

this.onItemFocus() Return: boolean

The method to trigger when an element (node/line) is to be selected from unchecked.

Parameters: ( id, type )

NameType / FormatDescription
idStringThe id of the element, unique identifier.
typeStringThere are two types of elements: 'node','line'.

this.onItemBlur() Return: boolean

The method to trigger when an element (node/line) is selected to be unchecked.

Parameters: ( id, type )

NameType / FormatDescription
idStringThe id of the element, unique identifier.
typeStringThere are two types of elements: 'node','line'.

this.onItemMark() Return: boolean

The method that is triggered when an element (node/line) is marked with a bold color.

Parameters: ( id, type, mark )

NameType / FormatDescription
idStringThe id of the element, unique identifier.
typeStringThere are two types of elements: 'node','line'.
markBooleanWhether to label. Yes true, otherwise false.

this.onItemAdd() Return: boolean

Triggered when an element (node/line/group lane/memo) is added.

Parameters: ( id, type, json )

NameType / FormatDescription
idStringThe id of the element, unique identifier.
typeStringThere are four types of elements: 'node','line','area'(lane),'memo'.
json[ NodeItem | LineItem | AreaItem ]Describe the corresponding structural data for the element. When the element is a node, it is NodeItem data; when the element is a line, it is LineItem data; when the element is a lane, it is AreaItem data.

this.onItemResize() Return: boolean

Triggered when an element (node/group lane/memo) is moved to a new location.

Parameters: ( id, type, width, height )

NameType / FormatDescription
idStringThe id of the element, unique identifier.
typeStringThere are four types of elements: 'node','line','area'(lane),'memo'.
widthNumberThe new width of the element, in pixels; the smallest 104pxelement when it is a node, and the smallest when the element is a grouped lane 200px.
heightNumberThe new height of the element, in pixels; the smallest 26pxelement when it is a node, and the smallest when the element is a grouped lane 100px.

this.onLineMove() Return: boolean

The method to trigger when moving the position of the midline of a polyline.

Parameters: ( id, M, sort )

NameType / FormatDescription
idStringThe id of the line, unique identifier.
MNumberThe new X coordinate of the middle fold line (when type='lr' the time) or a new y-coordinate (as type='tb'), in pixels.
MNumberThe sorting of the midline segments that can be moved is sequentially taken as 'Ml', 'M', 'Mr', and the default is 'M' when not filled.

this.onLineSetStyle() Return: boolean

The method to trigger when transforming the display style of a line.

Parameters: ( id, style )

NameType / FormatDescription
idStringThe id of the line, unique identifier.
styleStringThere are four new display style of line: 'direct' directed solid line, 'dirDashed' directed dashed line, 'segment' solid line and 'segDashed' dashed line.

this.onLineSetType() Return: boolean

The method to trigger when transforming the type of a line.

Parameters: ( id, type )

NameType / FormatDescription
idStringThe id of the line, unique identifier.
typeStringThere are three new types of line: 'sl'straight lines; 'lr'folding lines that can be moved left and right in the middle section ; straight lines that can move 'tb' up and down in the middle section.

this.onLinePointMove() Return: boolean

When a node connected to an edge of a line is transformed (that is, an endpoint is changed from connected node A to connected node B), the method is triggered.

Parameters: ( id, newStart, newEnd )

NameType / FormatDescription
idStringThe id of the line, unique identifier.
newStartStringThe new starting node's id; if it is null then the original endpoint is unchanged.
newEndStringThe new target node's id; if it is null then the original endpoint is unchanged.

this.onBatchAdd() Return: boolean

Triggered when multiple elements (node/line) are added batchly.

Parameters: ( nodes, lines )

NameType / FormatDescription
nodesJsonMapInformation about multiple nodes to be added.
In the JsonMap, the key is the id of the node, and the value is the node detailed information, which belongs to the NodeItem type data.
linesJsonMapInformation about multiple lines to be added.
In the JsonMap, the key is the id of the line, and the value is the line detailed information, which belongs to the LineItem type data.

lineIdsString[]An array of all the ids of the lines to be deleted.

Parameters: ( nodeIds, lineIds, moveX, moveY )

NameType / FormatDescription

moveYNumberThe vertical displacement of all elements relative to the original position, in pixels.
